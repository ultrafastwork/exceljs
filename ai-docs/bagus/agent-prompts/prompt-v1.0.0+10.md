# AI Agent Prompt: Implement PR #3020 HAN CELL Excel Files Support

## Objective
Your goal is to implement PR #3020: "Fix #3014: Add support for HAN CELL Excel files" to allow ExcelJS to seamlessly load files generated by HAN CELL (Korean spreadsheet software) without crashing.

## Context & Details
- **PR Description**: ExcelJS crashes when reading files created by HAN CELL because the XML tags use the `x:` namespace prefix (e.g. `x:worksheet`, `x:row`) instead of unprefixed tags, and certain properties in workbook/app/core properties are assumed to always be defined.
- **Detailed File**: Refer to [pr-3020.md](file:///d:/projects/exceljs/ai-docs/prs/pr-3020.md) for full context on the required changes.
- **Rules File**: Always adhere to the project rules in [.windsurfrules](file:///d:/projects/exceljs/.windsurfrules).

## Instructions

### 1. Strip Namespace Prefix in SAX Parser
- Update `lib/utils/parse-sax.js` to strip `x:` namespace prefixes while preserving standard namespace prefixes like `dc:`, `cp:`.

### 2. Null Checks in Excel Reader
- Modify `lib/xlsx/xlsx.js` to add null/undefined checks before accessing `workbook`, `appProperties`, and `coreProperties`.

### 3. Graceful XML Tag Handling
- Ensure `lib/xlsx/xform/core/core-xform.js` and `lib/xlsx/xform/simple/shared-strings-xform.js` ignore unknown XML tags instead of throwing errors.

### 4. Integration Test
- Add a new integration test with a HAN CELL exported spreadsheet file (or programmatically mock it) to verify it loads, reads the worksheet name, row count, and cell values successfully.

### 5. Verification
- Verify your changes using existing unit and integration tests (`pnpm test:unit` and `pnpm test:integration`).
- Run lint checks (`pnpm lint` and `npx eslint`) to ensure styling compliance.
